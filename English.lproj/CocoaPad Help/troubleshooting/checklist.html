Troubleshooting Checklist

Something gone screwy in CocoaPad? Before breaking out in a sweaty panic, try the following steps:

  1. Search the online help system for your "problem." It could be "a feature, not a bug" (in other words, a different way of doing things).
  2. Was there an error message? Did it tell you anything about the problem? Even if it didn't, you should write it down anyway.
  3. Quit CocoaPad and open it again. Did the problem go away?
  4. Check to see if the bug occurs in TextEdit (in your Applications folder). CocoaPad and other Mac OS X-exclusive programs, such as TextEdit, share the same text editing functions, which are maintained by Apple. Bugs in Apple's text editing functions are out of our control. Try contacting Apple and reporting the bug to them.
  5. Trash CocoaPad's preference file. Go to your home folder, open Library, open Preferences, then find a file named com.htw.cocoapad.plist. Drag this file to the trash. If the problem goes away, empty the trash. Otherwise, drag the file back to your preferences folder.
  6. Logout, log back in as yourself, then open CocoaPad. Sometimes, things get so crazy and jumbled that it's best to start over.
  7. As a last resort, delete CocoaPad, then reinstall it from the installer disk.

Okay, if you've followed those things, and you can't find the answer, go on to the next section.
