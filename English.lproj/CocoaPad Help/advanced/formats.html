All about file formats

When the word processor came along, they had a great feature that text editors didn't have: the ability to format documents with colors, fonts, etc. Unfortunately, the programmers who made all the word processors in the world never agreed on one way to save the formatted text. So, each word processor had its special way of saving formatted text. Hence, there was lots of confusion and frustration because people had to figure out what format to use, or figure out what word processor someone had, just to send them a document that had formatting.

However, there is one standardized formatting scheme that allows most of the benefits of word processor formats, though. It's called "Rich Text Format", and users of the Microsoft Word word processor have known this for years. Apple's TextEdit text editor/word processor saves its documents in Rich Text Format, and many word processors can read Rich Text.

Here's a rundown of the 5 formats CocoaPad can save and open:

  • CocoaPad documents (.cpd files) can have graphics and formatting, but they are not currently compatible with any other word processor.
  • Rich Text Format documents (.rtf files) can have formatting and are compatible with many word processors, but they cannot contain graphics.
  • Rich Text Format with Attachments documents (.rtfd files) can have graphics and formatting, and they are compatible with TextEdit. However, they are in a Mac OS X-specific format, and will only work on Mac OS X systems. This was added only for compatibility with TextEdit.
  • Microsoft Word documents (.doc files) can have formatting, and are fully compatible with Microsoft Word, yet you need Mac OS X 10.3 Panther to use this feature. However, not all Microsoft Word documents will work with CocoaPad, because CocoaPad does not support all Microsoft Word document features, such as tables, macros, and graphics. Still, CocoaPad can create and edit simple Word documents. Word documents are also compatible with TextEdit for Mac OS X 10.3 Panther, but TextEdit does not support all of Word's document features, like CocoaPad.
  • Plain text documents (.txt files) do not have any formatting or graphics at all, but they can be read by almost all text editors/word processors.
CocoaPad can open almost any document, but if CocoaPad does not understand what is inside the document, the document will appear as plain text. CocoaPad can also open regular documents as plain text, like CocoaPad and Rich Text documents. To do so, check the Ignore formatting commands checkbox in the preference panel. This is useful, for example, if you want to view the behind-the-scenes codes inside a document that tell programs (including CocoaPad) how to format the document.

CocoaPad has its own formatting scheme also. It's a variant of the Rich Text with Attachments format, but instead of using Mac OS X-only file packages, CocoaPad encodes the formatting into RTFD data.

The "D" in RTFD stands for "directory", since TextEdit pioneered the technology, and its RTFD files are actually file packages. File packages are folders that are treated as a regular file by Mac OS X.

You save your document into any one of these formats by doing the following:

  1. Pull down the Format menu.
  2. Select Convert to <format>, where <format> is the name of the format you want. If the format you want is grayed-out, then your document is already in that format.
If the Convert to Microsoft Word format format is always grayed-out, even if your document is not in Microsoft Word format, then this means you have Mac OS X version 10.2 Jaguar or earlier. You're operating system version needs to be at least Mac OS X 10.3 (Panther) in order to save and open Word documents. To purchase Mac OS X 10.3, or to learn more about it, visit Apple's Mac OS X website.
You can specify what format you want CocoaPad to use when it creates a new document in CocoaPad's preferences. By default, it creates CocoaPad documents.
